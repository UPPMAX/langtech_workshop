Running VSCode on compute nodes¶

It is possible to use the login node as a "jump box"

$ ssh -J host1 host2

We can also set this up in the ~/.ssh/config file as:

~/.ssh/config
### First jump host. Directly reachable
Host betajump
  HostName jumphost1.example.org

### Host to jump to via jumphost1.example.org
Host behindbeta
  HostName behindbeta.example.org
  ProxyJump  betajump

However, we don't know which compute node we will be allocated. This can be solved by some regex

bash title="~/.ssh/config Host *+* ProxyCommand ssh $(echo %h | sed 's/+[^+]*$//;s/\([^+%%]*\)%%\([^+]*\)$/\2 -l \1/;s/:/ -p /') nc $(echo %h | sed 's/^.*+//;/:/!s/$/ %p/;s/:/ /') 

which allows you to do stuff like
ssh matpiq@rackham5+rackham2

in VScode
